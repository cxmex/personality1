As of June 2026, the MCP ecosystem has hundreds of servers for US/European business tools (Slack, GitHub, Salesforce, Stripe) but zero connectors purpose-built for the Mexican business environment. 2 MCP Architecture Overview

2.1 How the Protocol Works

MCP is built on JSON-RPC 2.0, the same lightweight remote procedure call protocol used by Ethereum and the Language Server Protocol. Every message between client and server is a JSON object with a method name, parameters, and an ID for request-response correlation.

The protocol defines three participant roles:

Role Description Example
MCP Host The AI application that manages one or multiple MCP clients Claude Desktop, Claude Code, VS Code, ChatGPT
MCP Client A component within the host that maintains a connection to one MCP server Internal client instance created per server connection
MCP Server A program that exposes tools, resources, and prompts to MCP clients Your People Analytics server, a Facturacion server

2.2 The Three Primitives

MCP servers expose functionality through three core primitives:

Primitive Purpose Discovery Execution
Tools Executable functions the AI can invoke (query DB, call API, compute) tools/list tools/call
Resources Read-only data sources (file contents, DB records, API responses) resources/list resources/read
Prompts Reusable interaction templates (system prompts, few-shot examples) prompts/list prompts/get

2.3 Protocol Lifecycle

Every MCP session follows a structured lifecycle:

  1. Initialize: Client sends initialize with its capabilities and protocol version. Server responds with its capabilities (which primitives it supports, whether it sends change notifications).
  2. Capability Negotiation: Both sides declare what they support. A server might declare "tools": {"listChanged": true} and "resources": {}.
  3. Ready: Client sends notifications/initialized to signal it is ready.
  4. Operation: Client discovers tools via tools/list, then invokes them via tools/call with JSON arguments. Server returns structured content (text, images, resources).
  5. Notifications: Server can push notifications/tools/list_changed when available tools change dynamically.
  6. Termination: Client closes connection or sends DELETE to session endpoint.

2.4 Transport Mechanisms

Transport How It Works Use Case
stdio Client launches server as subprocess; messages via stdin/stdout Local servers (dev tools, filesystem access)
Streamable HTTP HTTP POST for client-to-server; optional SSE for streaming responses Remote servers (SaaS APIs, cloud deployments) — this is what we will use

The Streamable HTTP transport is the key for our use case. It allows a single MCP server deployed on Render to serve multiple Claude users simultaneously. The server exposes a single endpoint (e.g., https://mcp.personality1.com/mcp) that accepts POST requests with JSON-RPC messages and optionally returns SSE streams for long-running operations.

2.5 Authentication (OAuth 2.1)

MCP uses OAuth 2.1 for authentication on HTTP transports. The flow works as follows:

  1. Client sends request without token to MCP server
  2. Server returns 401 Unauthorized with WWW-Authenticate header pointing to Protected Resource Metadata
  3. Client fetches /.well-known/oauth-protected-resource from MCP server to discover authorization server URL
  4. Client fetches /.well-known/oauth-authorization-server from auth server for endpoints
  5. Client performs Dynamic Client Registration (RFC 7591) if needed
  6. Client redirects user to authorization URL with PKCE challenge
  7. After user consent, client exchanges authorization code for access token
  8. Client includes Authorization: Bearer <token> on all subsequent requests

For simpler deployments (our Tier 1 tools), we can use API key authentication as a pragmatic starting point, then migrate to full OAuth 2.1 when we publish to the MCP marketplace.

2.6 How Claude Code Connects to Custom MCP Servers

Adding a remote MCP server to Claude Code is a single command:

# Add a remote MCP server (Streamable HTTP)
claude mcp add --transport http people-analytics https://mcp.personality1.com/mcp

# Add a local MCP server (stdio)
claude mcp add --transport stdio people-analytics uv run people_analytics_mcp.py

# List configured servers
claude mcp list

# Test with the MCP Inspector
npx -y @modelcontextprotocol/inspector

Tool 4: Nomina MCP (Mexican Payroll)

Calculates Mexican payroll with full compliance to the Ley Federal del Trabajo. This is the most legally complex tool but also the most valuable — payroll errors in Mexico trigger IMSS/SAT audits.

calculate_payroll calculate_sdi calculate_aguinaldo calculate_vacaciones calculate_prima_vacacional calculate_ptu calculate_finiquito calculate_isr calculate_imss_cuotas generate_nomina_cfdi get_employee_cost simulate_raise

Key Calculations:

  • SDI (Salario Diario Integrado): Daily wage + proportional aguinaldo + proportional prima vacacional + other prestaciones. Formula: SDI = SD * (1 + aguinaldo_days/365 + (vacation_days * 0.25)/365)
  • ISR: Monthly income tax per SAT Art. 96 tables (updated annually)
  • IMSS Cuotas: Employee + employer contributions across 5 branches (Enfermedades y Maternidad, Invalidez y Vida, Retiro, Cesantia, Guarderias)
  • Aguinaldo: Minimum 15 days salary, prorated by days worked in year
  • Prima Vacacional: 25% of vacation days salary
  • PTU: 10% of company profits distributed to workers (capped at 3 months salary or average of last 3 years)

6 Technical Implementation Guide

6.1 MCP Server Skeleton in Python (FastMCP)

The Python MCP SDK provides FastMCP, a high-level framework that auto-generates tool definitions from Python type hints and docstrings. Here is the complete skeleton for the People Analytics MCP server:

# people_analytics_mcp.py
# MCP Server for Personality1 People Analytics

from contextlib import asynccontextmanager
from typing import Any, Optional
import httpx
from pydantic import BaseModel
from mcp.server.fastmcp import FastMCP, Context

# ========== CONFIGURATION ==========
SUPABASE_URL = "https://gbkhkbfbarsnpbdkxzii.supabase.co"
SUPABASE_KEY = "your-supabase-anon-key"  # Use env var in production
HEADERS = {
    "apikey": SUPABASE_KEY,
    "Authorization": f"Bearer {SUPABASE_KEY}",
    "Content-Type": "application/json",
}

# ========== LIFESPAN (DB CONNECTION POOL) ==========
@asynccontextmanager
async def lifespan(server: FastMCP):
    """Manage httpx client lifecycle."""
    async with httpx.AsyncClient(
        base_url=SUPABASE_URL,
        headers=HEADERS,
        timeout=30.0
    ) as client:
        yield {"db": client}

# ========== SERVER INSTANCE ==========
mcp = FastMCP(
    "People Analytics",
    lifespan=lifespan,
    dependencies=["httpx", "pydantic"],
)

# ========== TOOLS ==========

@mcp.tool()
async def get_candidates(
    position: Optional[str] = None,
    ctx: Context = None,
) -> str:
    """Get all candidates, optionally filtered by position.

    Args:
        position: Filter by position slug (almacen, ventas_mostrador,
                  pueblaventas). Leave empty for all candidates.
    """
    db: httpx.AsyncClient = ctx.request_context["db"]
    url = "/rest/v1/participants?select=*"
    if position:
        url += f"&position=eq.{position}"

    resp = await db.get(url)
    resp.raise_for_status()
    candidates = resp.json()

    if not candidates:
        return "No candidates found."

    lines = []
    for c in candidates:
        lines.append(
            f"- {c['name']} ({c['email']}) | Position: {c['position']}"
            f" | Date: {c['created_at'][:10]}"
        )
    return f"Found {len(candidates)} candidates:\n" + "\n".join(lines)


@mcp.tool()
async def get_disc_profile(
    candidate_email: str,
    ctx: Context = None,
) -> str:
    """Get DISC personality profile for a specific candidate.

    Args:
        candidate_email: The email address of the candidate.
    """
    db: httpx.AsyncClient = ctx.request_context["db"]

    # Get participant ID
    resp = await db.get(
        f"/rest/v1/participants?email=eq.{candidate_email}&select=id,name"
    )
    participants = resp.json()
    if not participants:
        return f"No candidate found with email {candidate_email}"

    pid = participants[0]["id"]
    name = participants[0]["name"]

    # Get DISC results
    resp = await db.get(
        f"/rest/v1/results?participant_id=eq.{pid}"
        f"&test_type=eq.disc&select=*"
    )
    results = resp.json()
    if not results:
        return f"No DISC results found for {name}"

    scores = results[0]["scores"]
    dominant = max(scores, key=scores.get)

    profile_desc = {
        "D": "Dominance - Direct, decisive, competitive",
        "I": "Influence - Enthusiastic, persuasive, optimistic",
        "S": "Steadiness - Patient, reliable, team-oriented",
        "C": "Conscientiousness - Analytical, precise, quality-focused",
    }

    return (
        f"DISC Profile for {name}:\n"
        f"  D (Dominance):      {scores.get('D', 0)}\n"
        f"  I (Influence):       {scores.get('I', 0)}\n"
        f"  S (Steadiness):      {scores.get('S', 0)}\n"
        f"  C (Conscientiousness): {scores.get('C', 0)}\n\n"
        f"Dominant Type: {dominant} - {profile_desc[dominant]}"
    )


@mcp.tool()
async def compare_candidates(
    email_1: str,
    email_2: str,
    ctx: Context = None,
) -> str:
    """Compare two candidates across all assessment dimensions.

    Args:
        email_1: Email of the first candidate.
        email_2: Email of the second candidate.
    """
    db: httpx.AsyncClient = ctx.request_context["db"]

    async def get_all_results(email):
        resp = await db.get(
            f"/rest/v1/participants?email=eq.{email}&select=id,name"
        )
        parts = resp.json()
        if not parts:
            return None, None
        pid = parts[0]["id"]
        name = parts[0]["name"]
        resp = await db.get(
            f"/rest/v1/results?participant_id=eq.{pid}&select=*"
        )
        return name, resp.json()

    name1, results1 = await get_all_results(email_1)
    name2, results2 = await get_all_results(email_2)

    if not name1:
        return f"Candidate {email_1} not found."
    if not name2:
        return f"Candidate {email_2} not found."

    comparison = f"=== Candidate Comparison ===\n\n"
    comparison += f"Candidate A: {name1}\nCandidate B: {name2}\n\n"

    tests_1 = {r["test_type"]: r["scores"] for r in results1}
    tests_2 = {r["test_type"]: r["scores"] for r in results2}

    all_tests = set(tests_1.keys()) | set(tests_2.keys())
    for test in sorted(all_tests):
        comparison += f"--- {test.upper()} ---\n"
        s1 = tests_1.get(test, {})
        s2 = tests_2.get(test, {})
        for key in sorted(set(s1.keys()) | set(s2.keys())):
            comparison += f"  {key}: A={s1.get(key, 'N/A')} vs B={s2.get(key, 'N/A')}\n"
        comparison += "\n"

    return comparison


@mcp.tool()
async def get_team_composition(ctx: Context = None) -> str:
    """Get the current team DISC composition and balance analysis.
    Shows the distribution of personality types across all hired team members.
    """
    db: httpx.AsyncClient = ctx.request_context["db"]

    # Fetch all DISC results
    resp = await db.get(
        "/rest/v1/results?test_type=eq.disc&select=*,participants(*)"
    )
    results = resp.json()
    if not results:
        return "No DISC results found in the system."

    totals = {"D": 0, "I": 0, "S": 0, "C": 0}
    dominant_counts = {"D": 0, "I": 0, "S": 0, "C": 0}

    for r in results:
        scores = r["scores"]
        for dim in totals:
            totals[dim] += scores.get(dim, 0)
        dominant = max(scores, key=scores.get)
        dominant_counts[dominant] += 1

    n = len(results)
    output = f"Team DISC Composition ({n} members):\n\n"
    for dim in ["D", "I", "S", "C"]:
        avg = totals[dim] / n
        output += f"  {dim}: avg={avg:.1f} | {dominant_counts[dim]} dominant\n"

    return output

# facturacion_mcp.py
# MCP Server for Mexican CFDI Electronic Invoicing

import os
from contextlib import asynccontextmanager
from typing import Optional
import httpx
from mcp.server.fastmcp import FastMCP, Context

# ========== FACTURAPI CONFIGURATION ==========
FACTURAPI_KEY = os.environ.get("FACTURAPI_KEY", "")
FACTURAPI_BASE = "https://www.facturapi.io/v2"

@asynccontextmanager
async def lifespan(server: FastMCP):
    async with httpx.AsyncClient(
        base_url=FACTURAPI_BASE,
        headers={
            "Authorization": f"Bearer {FACTURAPI_KEY}",
            "Content-Type": "application/json",
        },
        timeout=30.0,
    ) as client:
        yield {"api": client}

mcp = FastMCP("Facturacion Mexico", lifespan=lifespan)


@mcp.tool()
async def create_cfdi(
    customer_rfc: str,
    customer_name: str,
    customer_zip: str,
    customer_tax_regime: str,
    cfdi_use: str,
    items: list[dict],
    payment_form: str = "01",
    payment_method: str = "PUE",
    ctx: Context = None,
) -> str:
    """Create a CFDI 4.0 electronic invoice (factura).

    Args:
        customer_rfc: Customer's RFC (tax ID), e.g. 'XAXX010101000'.
        customer_name: Customer's legal name as registered with SAT.
        customer_zip: Customer's fiscal zip code (domicilio fiscal).
        customer_tax_regime: SAT tax regime code, e.g. '601' for General.
        cfdi_use: CFDI use code, e.g. 'G03' for Gastos en general.
        items: List of line items. Each item needs: description, product_key
               (SAT catalog), quantity, unit_price, tax_included (bool).
               Example: [{"description": "Servicio de consultoria",
               "product_key": "80101500", "quantity": 1,
               "unit_price": 5000.00, "tax_included": true}]
        payment_form: SAT payment form code. '01'=Efectivo, '03'=Transferencia,
                      '04'=Tarjeta credito, '28'=Tarjeta debito.
        payment_method: 'PUE' for single payment, 'PPD' for installments.
    """
    api: httpx.AsyncClient = ctx.request_context["api"]

    invoice_data = {
        "customer": {
            "legal_name": customer_name,
            "tax_id": customer_rfc,
            "tax_system": customer_tax_regime,
            "address": {"zip": customer_zip},
        },
        "items": [
            {
                "description": item["description"],
                "product_key": item["product_key"],
                "quantity": item["quantity"],
                "price": item["unit_price"],
                "tax_included": item.get("tax_included", True),
            }
            for item in items
        ],
        "use": cfdi_use,
        "payment_form": payment_form,
        "payment_method": payment_method,
    }

    resp = await api.post("/invoices", json=invoice_data)
    if resp.status_code != 200:
        return f"Error creating CFDI: {resp.status_code} - {resp.text}"

    invoice = resp.json()
    return (
        f"CFDI created successfully!\n"
        f"  UUID: {invoice['uuid']}\n"
        f"  Folio: {invoice['folio_number']}\n"
        f"  Total: ${invoice['total']:,.2f} MXN\n"
        f"  Status: {invoice['status']}\n"
        f"  PDF: {invoice.get('pdf_url', 'N/A')}"
    )


@mcp.tool()
async def cancel_cfdi(
    invoice_id: str,
    cancellation_reason: str = "02",
    ctx: Context = None,
) -> str:
    """Cancel a previously issued CFDI.

    Args:
        invoice_id: The FacturAPI invoice ID or UUID.
        cancellation_reason: SAT cancellation code. '01'=CFDI with errors
            with relation, '02'=CFDI with errors without relation,
            '03'=Operation not performed, '04'=Nominative operation.
    """
    api: httpx.AsyncClient = ctx.request_context["api"]
    resp = await api.delete(
        f"/invoices/{invoice_id}",
        params={"motive": cancellation_reason},
    )
    if resp.status_code != 200:
        return f"Error cancelling CFDI: {resp.text}"
    return f"CFDI {invoice_id} cancelled successfully."


@mcp.tool()
async def get_tax_summary(
    month: int,
    year: int,
    ctx: Context = None,
) -> str:
    """Get tax summary for a given month (IVA, ISR retention totals).

    Args:
        month: Month number (1-12).
        year: Year (e.g. 2026).
    """
    api: httpx.AsyncClient = ctx.request_context["api"]
    resp = await api.get(
        "/invoices",
        params={
            "date[gte]": f"{year}-{month:02d}-01",
            "date[lte]": f"{year}-{month:02d}-31",
            "status": "valid",
        },
    )
    invoices = resp.json().get("data", [])

    total_subtotal = sum(i.get("subtotal", 0) for i in invoices)
    total_iva = sum(i.get("total", 0) - i.get("subtotal", 0) for i in invoices)
    total = sum(i.get("total", 0) for i in invoices)

    return (
        f"Tax Summary for {month:02d}/{year}:\n"
        f"  Invoices issued: {len(invoices)}\n"
        f"  Subtotal: ${total_subtotal:,.2f} MXN\n"
        f"  IVA collected: ${total_iva:,.2f} MXN\n"
        f"  Total invoiced: ${total:,.2f} MXN"
    )

def calc_isr_mensual(ingreso_gravable: float) -> float:
    """Calculate monthly ISR using Art. 96 tables."""
    for lim_inf, lim_sup, cuota_fija, tasa in ISR_TABLE:
        if lim_inf <= ingreso_gravable <= lim_sup:
            excedente = ingreso_gravable - lim_inf
            return cuota_fija + (excedente * tasa)
    return 0.0


@mcp.tool()
def calculate_sdi(
    salario_diario: float,
    aguinaldo_days: int = 15,
    vacation_days: int = 12,
    prima_vacacional_pct: float = 0.25,
) -> str:
    """Calculate Salario Diario Integrado (SDI) for IMSS registration.

    Args:
        salario_diario: Daily wage in MXN.
        aguinaldo_days: Days of aguinaldo (minimum 15 by law).
        vacation_days: Days of vacation per year (depends on seniority).
        prima_vacacional_pct: Prima vacacional percentage (minimum 25%).
    """
    factor = 1 + (aguinaldo_days / 365) + ((vacation_days * prima_vacacional_pct) / 365)
    sdi = salario_diario * factor

    return (
        f"SDI Calculation:\n"
        f"  Salario Diario: ${salario_diario:,.2f}\n"
        f"  Factor de integracion: {factor:.6f}\n"
        f"  Aguinaldo component: {aguinaldo_days}/365 = {aguinaldo_days/365:.6f}\n"
        f"  Prima vac component: {vacation_days}*{prima_vacacional_pct}/365 = "
        f"{(vacation_days*prima_vacacional_pct)/365:.6f}\n"
        f"  SDI: ${sdi:,.2f}\n\n"
        f"  Monthly IMSS base: ${sdi * 30:,.2f}"
    )


@mcp.tool()
def calculate_payroll(
    salario_mensual: float,
    aguinaldo_days: int = 15,
    vacation_days: int = 12,
) -> str:
    """Calculate complete monthly payroll for one employee.

    Shows gross pay, ISR, IMSS employee contribution, and net pay.
    Compliant with Mexican Ley Federal del Trabajo.

    Args:
        salario_mensual: Monthly gross salary in MXN.
        aguinaldo_days: Annual aguinaldo days (min 15).
        vacation_days: Annual vacation days (by seniority).
    """
    sd = salario_mensual / 30
    factor = 1 + (aguinaldo_days / 365) + ((vacation_days * 0.25) / 365)
    sdi = sd * factor

    # ISR
    isr = calc_isr_mensual(salario_mensual)

    # IMSS Employee contributions (approximate rates)
    uma_mensual = 3424.20  # UMA 2026 approximate
    imss_employee = sdi * 30 * 0.02375  # ~2.375% employee total

    # IMSS Employer contributions (approximate rates)
    imss_employer = sdi * 30 * 0.13915  # ~13.915% employer total

    # Infonavit (employer only)
    infonavit = sdi * 30 * 0.05

    # SAR (employer only)
    sar = sdi * 30 * 0.02

    neto = salario_mensual - isr - imss_employee
    costo_total = salario_mensual + imss_employer + infonavit + sar

    return (
        f"=== NOMINA MENSUAL ===\n\n"
        f"Salario Bruto:      ${salario_mensual:>12,.2f}\n"
        f"(-) ISR:            ${isr:>12,.2f}\n"
        f"(-) IMSS empleado:  ${imss_employee:>12,.2f}\n"
        f"━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━\n"
        f"NETO A PAGAR:       ${neto:>12,.2f}\n\n"
        f"=== COSTO PATRONAL ===\n"
        f"Salario:            ${salario_mensual:>12,.2f}\n"
        f"(+) IMSS patron:    ${imss_employer:>12,.2f}\n"
        f"(+) INFONAVIT:      ${infonavit:>12,.2f}\n"
        f"(+) SAR:            ${sar:>12,.2f}\n"
        f"━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━\n"
        f"COSTO TOTAL:        ${costo_total:>12,.2f}\n\n"
        f"SDI: ${sdi:,.2f} | Factor: {factor:.4f}"
    )


@mcp.tool()
def calculate_aguinaldo(
    salario_diario: float,
    dias_trabajados: int = 365,
    aguinaldo_days: int = 15,
) -> str:
    """Calculate aguinaldo (Christmas bonus) with ISR exemption.

    Args:
        salario_diario: Daily wage in MXN.
        dias_trabajados: Days worked in the year (for proration).
        aguinaldo_days: Company's aguinaldo days (minimum 15).
    """
    proporcional = (dias_trabajados / 365) * aguinaldo_days
    bruto = salario_diario * proporcional

    # ISR exemption: 30 UMA dias
    uma_diaria = 114.14  # UMA 2026 approximate
    exento = uma_diaria * 30
    gravable = max(0, bruto - exento)

    return (
        f"=== CALCULO DE AGUINALDO ===\n"
        f"  Salario diario: ${salario_diario:,.2f}\n"
        f"  Dias trabajados: {dias_trabajados}\n"
        f"  Dias proporcionales: {proporcional:.2f}\n"
        f"  Aguinaldo bruto: ${bruto:,.2f}\n"
        f"  Exento (30 UMAs): ${exento:,.2f}\n"
        f"  Gravable para ISR: ${gravable:,.2f}"
    )


@mcp.tool()
def simulate_raise(
    current_salary: float,
    raise_percentage: float,
) -> str:
    """Simulate the total cost impact of giving an employee a raise.

    Shows the before/after comparison including all employer obligations.

    Args:
        current_salary: Current monthly gross salary in MXN.
        raise_percentage: Proposed raise as percentage (e.g., 10 for 10%).
    """
    new_salary = current_salary * (1 + raise_percentage / 100)

    def total_cost(sal):
        sd = sal / 30
        sdi = sd * 1.0493  # Approx factor for 15 days + 12 vac
        employer = sdi * 30 * (0.13915 + 0.05 + 0.02)
        return sal + employer

    cost_before = total_cost(current_salary)
    cost_after = total_cost(new_salary)
    delta = cost_after - cost_before

    return (
        f"=== SIMULACION DE AUMENTO ===\n"
        f"  Salario actual:    ${current_salary:>10,.2f}\n"
        f"  Aumento ({raise_percentage}%):    ${new_salary - current_salary:>10,.2f}\n"
        f"  Salario nuevo:     ${new_salary:>10,.2f}\n\n"
        f"  Costo total antes: ${cost_before:>10,.2f}\n"
        f"  Costo total nuevo: ${cost_after:>10,.2f}\n"
        f"  Incremento real:   ${delta:>10,.2f} (+{delta/cost_before*100:.1f}%)"
    )
